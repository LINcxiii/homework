# 花卉特征分类系统 技术报告

## 一、项目背景
随着计算机视觉与智能分类技术的快速普及，图像特征分类技术已广泛应用于农林识别、智能检测、物种辨识等多个领域，成为轻量化人工智能落地应用的核心技术之一。花卉物种精准分类是农林科普、园艺培育、植物资源统计场景的重要基础工作，传统人工分类方式效率低下、主观性强、容错率低，而现有智能分类方案多依赖复杂的深度学习框架、海量图像数据集与专业运行环境，存在部署门槛高、代码结构混乱、可复用性差、拓展难度大等问题，难以适配小型轻量化、快速落地的花卉分类应用场景。

当前市面上多数小型分类程序存在功能碎片化、架构不规范的问题，零散的代码逻辑无法实现数据处理、模型训练、推理预测、数据管理的全流程闭环，难以支撑二次开发与功能迭代。而百度飞桨等主流AI开源框架具备标准化、工程化的项目分层架构，其完整的数据集处理、模型训练、参数迭代、推理部署的流程体系，为轻量化分类项目的架构设计提供了成熟的参考范式。

基于以上行业现状与现存技术痛点，本次项目借鉴主流AI工程的标准化分层设计思想，依托原生Python开发一套轻量化、零依赖、可拓展的花卉特征分类系统。项目摒弃传统复杂的框架依赖与繁琐的环境配置，在保留大型智能分类项目完整工程架构的基础上，精简技术栈、优化代码逻辑，实现从数据集构建、模型训练、参数持久化到智能分类、实验记录管理的全流程自动化。系统兼顾实用性与工程性，既能够高效完成花卉特征精准分类的核心业务需求，也可作为小型智能分类项目的通用基础模板，为轻量化AI分类项目的快速开发、迭代落地提供标准化参考方案。

## 二、需求分析
### 功能需求
1. 完成数据集的自动化构建，生成多类别样本特征数据，作为模型训练的数据源；
2. 实现模型训练逻辑，完成样本数据拟合，同时支持模型参数本地保存、本地加载复用；
3. 实现样本推理功能，输入一组特征数据，程序自动完成运算并输出对应的花卉类别；
4. 实现推理记录的内存存储、历史记录查看、记录本地持久化保存三项数据管理功能。

### 非功能需求
整体代码采用面向对象模块化拆分，不同功能划分至独立类与函数内，结构清晰；无第三方库依赖，在主流Windows系统下均可稳定运行；采用中文指令交互模式，操作简单易懂，程序运行稳定无崩溃问题。

## 三、核心模块逻辑说明
整个项目按照分层思想拆分为六大核心模块，整体分层逻辑参考飞桨图像分类项目的分层方式：

1. 数据集模块：FlowerDataset类对应数据读取与生成模块，负责批量生成多类别样本特征，构建完整的训练数据集，为后续模型训练提供数据支撑。
2. 模型主体模块：ClassifyModel为整个项目的核心模块，内部封装训练、推理、参数保存、参数加载四类核心方法。训练阶段会依据输入的数据集生成专属的权重参数；推理阶段依靠已训练完成的参数完成特征匹配，输出分类结果；同时可以将权重保存为json文件，后续可直接读取文件恢复训练好的模型，不用重复训练。
3. 模型训练模块：调用模型内部训练方法，完成全部样本的拟合过程，生成稳定可用的模型参数。
4. 参数持久化模块：将内存中的模型参数写入本地文件，或是从本地文件读取参数恢复模型状态，适合多次重复实验的场景使用。
5. 样本推理模块：接收外部输入的特征序列，调用模型推理方法得到分类结果，完成整个分类业务流程。
6. 交互与记录管理模块：搭建指令交互入口，统一调度各个底层模块协同工作；同时缓存每一次的推理结果，支持记录查看与文件导出，方便后续实验结果整理复盘。

## 四、系统测试结果
1. 环境兼容性测试：在Python3.11环境下可正常启动运行，不存在语法报错、依赖缺失等问题，适配性良好；
2. 模型训练测试：执行训练指令后可顺利完成全部数据拟合，模型参数可正常保存为model.json文件，同时可以正常加载本地参数完成模型恢复；
3. 分类推理测试：输入多组不同的特征数据，系统均可以稳定输出对应的花卉品类，分类逻辑运行稳定；
4. 记录管理功能测试：多次推理产生的记录可以完整保存、有序展示，同时可以整体导出至本地文本文件，数据读写正常无丢失。

整体各项功能均达到前期设计目标，完整实现预期的分类工程需求。
